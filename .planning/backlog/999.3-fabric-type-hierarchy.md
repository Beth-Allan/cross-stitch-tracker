# Fabric Type Hierarchy

**What:** Replace the hardcoded fabric type dropdown with a hierarchical/manageable type system. Currently the type is a flat list (Aida, Linen, Lugana, Jobelan, Evenweave, etc.) but types have parent-child relationships.

**User context:** Lugana and Jobelan are both evenweaves. Linen has sub-types (Edinburgh, Belfast, etc.). The user wants to track specific fabric types, not just broad categories.

**Options to explore:**
1. Fabric types as a separate DB table with optional parent_id (tree structure)
2. Two-level dropdown: category (Evenweave, Linen, Aida) → sub-type (Lugana, Jobelan, Edinburgh, Belfast)
3. Free-text with autocomplete seeded from known types

**Why post-MVP:** Flat dropdown with "Other" works for MVP catalog building. Hierarchy matters once the collection grows and filtering/grouping by type becomes useful.
