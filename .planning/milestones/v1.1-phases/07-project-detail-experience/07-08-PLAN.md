The SearchToAdd component currently calls `getThreads(undefined, undefined, search || undefined)` for threads, ignoring the already-supported `colorFamily` parameter. Users reported the search is "effed up" partly because there's no color dropdown to narrow results when searching 500+ DMC threads.

**Add optional colorFamily filter:**

1. Add a `colorFamily` state variable initialized to empty string (all colors):
```typescript
const [colorFamily, setColorFamily] = useState("");
```

2. Add `colorFamily` to the dependency array of the fetch useEffect (alongside `search` and `supplyType`).

3. In the fetchItems function, pass `colorFamily` to `getThreads` only when `supplyType === "thread"`:
```typescript
if (supplyType === "thread") {
  results = await getThreads(undefined, colorFamily || undefined, search || undefined);
}
```

4. Add a color family dropdown ONLY when `supplyType === "thread"`. Render it between the search input and the results list. Use a native `<select>` element for simplicity (no need for a shadcn Select inside an already-complex floating panel):

```tsx
{supplyType === "thread" && (
  <div className="border-border border-b px-2 pb-2">
    <select
      value={colorFamily}
      onChange={(e) => setColorFamily(e.target.value)}
      className="border-border bg-card text-foreground w-full rounded border px-2 py-1.5 text-sm"
      aria-label="Filter by color family"
    >
      <option value="">All colors</option>
      <option value="RED">Red</option>
      <option value="ORANGE">Orange</option>
      <option value="YELLOW">Yellow</option>
      <option value="GREEN">Green</option>
      <option value="BLUE">Blue</option>
      <option value="PURPLE">Purple</option>
      <option value="BROWN">Brown</option>
      <option value="BLACK">Black</option>
      <option value="WHITE">White</option>
      <option value="GRAY">Gray</option>
      <option value="NEUTRAL">Neutral</option>
    </select>
  </div>
)}
```

Place this JSX element after the search input `<div className="p-2">` block and before the `<div className="border-border max-h-72 overflow-y-auto border-t">` results block.

5. Reset `colorFamily` to `""` when the component unmounts or when `supplyType` changes (already handled by component remount since SearchToAdd is conditionally rendered per section type).

**Important:** The `<option>` values MUST match the `ColorFamily` enum exactly (all caps) because `getThreads` passes the value directly to Prisma's where clause: `where.colorFamily = colorFamily`.
