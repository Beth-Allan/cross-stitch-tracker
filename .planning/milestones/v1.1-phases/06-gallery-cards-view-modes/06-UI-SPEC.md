## Typography

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Heading | Fraunces | 24px (text-2xl) | 600 (semibold) | 1.2 | "Project Gallery" page heading |

Only two weights are used across the entire phase: 400 (regular) and 600 (semibold).

## Visual Focal Point

The primary visual anchor on gallery cards is the **cover image** (or status-colored gradient placeholder) combined with the **status badge overlay** positioned at top-left. These two elements establish the card's identity and state at a glance. The **project name link** is the primary interaction target -- it is the only clickable element on the card and draws the eye through its Fraunces heading font and emerald hover underline.

## Layout Specifications

### Gallery View Grid

```
grid-template-columns: repeat(auto-fill, minmax(280px, 340px))
gap: 24px
```

Responsive behavior: auto-fill naturally reduces columns as viewport shrinks.
- 1280px+: 3-4 cards per row
- 768px-1279px: 2-3 cards per row
- <768px: 1-2 cards per row (cards stretch to fill)

### Gallery Card Structure

```
[Card container: rounded-xl, overflow-hidden, border, hover:-translate-y-1]
  [Cover image area: aspect-[4/3], relative, overflow-hidden]
    [Image or CoverPlaceholder]
    [Gradient overlay: h-12, from-black/20 on real images]
    [StatusBadge: absolute top-3 left-3]
    [Size badge: absolute top-3 right-3]
  [Card body: p-4, flex-col, gap-1.5, flex-1]
    [Project name: Fraunces 14px semibold, line-clamp-2, hover:emerald]
    [Designer: Source Sans 14px regular, truncate]
    [Stitch count (non-WIP only): Source Sans 12px]
    [Genre tags: max 3 + overflow pill]
    [Spacer: flex-1 min-h-1]
    [Status-specific footer]
```

### List View Row

```
grid-template-columns: 8px minmax(180px, 2fr) minmax(120px, 1fr) minmax(100px, 120px) 64px 56px
gap: 0 16px
padding: 8px 16px
align-items: center
```

Columns: status dot | name+designer | context stats | progress/kitting/date | status label | size badge

### Table View

Full-width table with columns:
- Always visible: Project (name), Status
- Hidden max-md: Designer
- Hidden max-sm: Size, Progress
- Hidden max-lg: Stitches, Colours, Time/Days
- Hidden max-xl: Activity

Table headers are sortable (clickable) with ChevronUp/ChevronDown indicators.

### Filter Bar Layout

```
Row 1: [Search input (flex: 0 1 280px)] [Status dropdown] [Size dropdown]
Row 2 (if active filters): [FilterChip] [FilterChip] ... [Clear all link]
Separator: border-b between filter area and content
Row 3: ["{N} projects" label] ... [Sort dropdown] [Cards|List|Table toggle]
```

### Page Layout

```
[Page header: "Project Gallery" heading + subtitle]
  gap: 8px (heading to subtitle)
[Filter bar]
  gap: 16px (internal between rows)
[Separator: border-border]
[Toggle bar: count + sort + view mode]
  gap: 24px (filter bar to toggle bar)
[Gallery grid / List / Table]
  gap: 24px (toggle bar to content)
```

## Interaction Specifications

### Card Hover

- Card: `hover:shadow-lg hover:shadow-stone-900/8 hover:-translate-y-1` with `transition-all duration-200`
- Cover image: `group-hover:scale-[1.03]` with `transition-transform duration-300`
- Project name: underline changes from `decoration-stone-300` to `decoration-emerald-500`, text changes to `text-emerald-700`

### Card Click

- Entire card is NOT clickable (no onClick on card container)
- Project name is a Link to `/charts/[id]` (existing detail page)
- Name styled as underlined button/link with Fraunces font

### View Mode Toggle

- Three-button segmented control: Cards (LayoutGrid icon) | List (List icon) | Table (Table2 icon)
- Active state: `bg-white dark:bg-stone-700 text-stone-900 shadow-sm`
- Inactive state: `text-stone-400 dark:text-stone-500 hover:text-stone-600`
- Container: `bg-stone-100 dark:bg-stone-800 rounded-lg p-0.5`
- Each button has a tooltip on hover describing the view
- Each button includes `<span className="sr-only">{mode} view</span>` for screen readers (e.g. "Cards view", "List view", "Table view")

### Sort Dropdown

- Trigger: same visual style as filter dropdowns (border, rounded-lg, text-sm)
- Options: Date Added, Name, Designer, Status, Size, Stitch Count
- Sort direction toggle: clicking same field flips asc/desc
- Default: Date Added, descending (newest first) per D-13
- Shared state between all views per D-11
- Table view: column headers ALSO trigger sort (in addition to dropdown)

### Filter Dropdowns

- Status: multi-select with checkboxes (emerald check when selected)
- Size: multi-select with checkboxes
- When filter has selections: border turns emerald-300, bg turns emerald-50, count badge appears
- Dropdown panel: absolute positioned below trigger, z-30, rounded-lg, shadow-lg, max-h-64 overflow-y-auto

### Filter Chips

- Appear below filter bar when any filter is active per D-03
- Each chip: `bg-stone-100 text-stone-700 border border-stone-200 rounded-full px-2 py-1`
- Dismiss (X): 3x3 icon, hover brightens text
- "Clear all" link at end: `text-stone-500 hover:text-stone-700`

### Search Input

- Left icon: Search (4x4, stone-400)
- Placeholder: "Search projects..."
- Focus: border changes to emerald-400
- Debounce: 300ms before filtering (client-side for responsiveness)
- Searches project name (match against chart name field)

### URL State Persistence (D-12)

All view state persisted in URL search params:
- `?view=gallery|list|table`
- `&sort=dateAdded|name|designer|status|size|stitchCount`
- `&dir=asc|desc`
- `&search=...`
- `&status=IN_PROGRESS,FINISHED` (comma-separated)
- `&size=Medium,Large` (comma-separated)

Refresh-safe, back-button friendly. Default when no params: `view=gallery&sort=dateAdded&dir=desc`.

## State Architecture

### URL Search Params (source of truth for view state)

```typescript
interface GallerySearchParams {
  view: 'gallery' | 'list' | 'table'    // default: 'gallery'
  sort: SortField                         // default: 'dateAdded'
  dir: 'asc' | 'desc'                    // default: 'desc'
  search: string                          // default: ''
  status: ProjectStatus[]                 // default: [] (all)
  size: SizeCategory[]                    // default: [] (all)
}
```

### Data Flow

```
Server Component (page.tsx)
  -> Fetch charts with supply counts (extended getCharts query)
  -> Resolve presigned image URLs
  -> Pass data + URL params to ProjectGallery client component

Client Component (ProjectGallery)
  -> Parse URL params for initial state
  -> Client-side filtering + sorting (no server round-trip)
  -> Update URL params on state change via router.push/replace
  -> Render FilterBar + ViewToggleBar + GalleryGrid
```

### Gallery Card Data Shape (derived from existing query + extensions)

Supply counts are computed from junction table `_count` in the extended query:
- `threadColourCount`: `project._count.threads`
- `beadTypeCount`: `project._count.beads`
- `specialtyItemCount`: `project._count.specialtyItems`
- `fabricStatus`: check `project.fabric !== null` -> 'fulfilled' : 'needed'
- `threadStatus`: `threadColourCount > 0` -> 'fulfilled' : check if chart expects threads (always yes) -> 'needed'
- `beadsStatus`: `beadTypeCount > 0` -> 'fulfilled' : `beadTypeCount === 0 && chartUsesBeads` -> 'needed' : 'not-applicable'
- `specialtyStatus`: same logic as beads

## Accessibility

### Keyboard Navigation

- Filter dropdowns: Enter/Space to open, Escape to close, arrow keys to navigate options
- View toggle: Tab between buttons, Enter/Space to activate
- Sort dropdown: same as filter dropdowns
- Card project names: Tab-focusable links with visible focus ring
- Table headers: Tab-focusable sort buttons
- Filter chips: Tab to chip, Enter on X to dismiss

### Screen Reader

- Status badges include text labels (not color-only)
- Kitting dots have `title` tooltips for hover AND `aria-label` for screen readers
- View mode buttons have `aria-pressed` or equivalent active indication AND `<span className="sr-only">{mode} view</span>` text (e.g. "Cards view", "List view", "Table view")
- Filter count announces total and filtered count
- Sort direction announced in button label (e.g. "Sort by name, ascending")
- Gallery grid uses `role="list"` with cards as `role="listitem"` (gallery view)

### Focus Management

- Focus ring: ring-2 ring-ring (emerald) on all interactive elements
- Cover images are decorative (`alt` describes project name for gallery context, empty alt for placeholder)
- Search input auto-focus not enabled (user navigates to page, may not want to search immediately)
