```typescript
interface ProgressBreakdownTabProps {
  buckets: ProgressBucket[];
  imageUrls: Record<string, string>;
}
```
Sort options (type `ProgressSortOption`):
- "closestToDone" → label "Closest to Done"
- "furthestFromDone" → label "Furthest from Done"
- "mostStitchingDays" → label "Most Stitching Days"
- "fewestStitchingDays" → label "Fewest Stitching Days"
- "recentlyStitched" → label "Recently Stitched"

Sorting logic within each bucket (client-side, `useMemo`):
- closestToDone: sort by progressPercent DESC
- furthestFromDone: sort by progressPercent ASC
- mostStitchingDays: sort by stitchingDays DESC
- fewestStitchingDays: sort by stitchingDays ASC
- recentlyStitched: sort by lastSessionDate DESC, null last

**Stacked bar chart** (pure CSS, no charting library):
- Container: `flex h-6 w-full overflow-hidden rounded-full bg-muted`
- Segments: `flex` items with width = `${(bucket.count / total) * 100}%`, `min-width: 24px` when count > 0 (per UI-SPEC spacing exception)
- Bucket accent colors (per UI-SPEC):
  - unstarted: `bg-stone-300`
  - 0-25: `bg-amber-400`
  - 25-50: `bg-emerald-400`
  - 50-75: `bg-sky-400`
  - 75-100: `bg-violet-400`
- Each segment gets `title="${bucket.label}: ${bucket.count} projects"` for tooltip per UI-SPEC accessibility

**Legend**: Inline `flex flex-wrap gap-3` below the bar with colored dots + labels

**Accordion sections**: Each bucket is a collapsible section (per D-13):
- **All collapsed by default**
- Header: tinted background matching bucket color (e.g., `bg-amber-50` for 0-25), chevron icon, label, count badge, range
- Chevron: rotates `-90deg` when collapsed, `0deg` when expanded. `transition-transform duration-200`
- Toggle via `useState<Set<ProgressBucketId>>` (empty set = all collapsed)
- Expanded content: list of BucketProjectRow components
- **Pagination**: Show 10 per page. If bucket has >10 projects, show "Show more (N remaining)" link.

Sort dropdown: `<select>` element following the exact pattern from WhatsNextTab (see PATTERNS.md), right-aligned above the stacked bar

```typescript
interface FinishedTabProps {
  projects: FinishedProjectData[];
  imageUrls: Record<string, string>;
}
```

Sort options (type `FinishedSortOption`):
- "finishDate" → label "Finish Date" (default, per D-15)
- "startToFinish" → label "Duration"
- "stitchCount" → label "Stitch Count"
- "stitchingDays" → label "Stitching Days"

Sorting logic (client-side, `useMemo`):
- finishDate: sort by finishDate DESC (most recent first), null last
- startToFinish: sort by startToFinishDays DESC, null last
- stitchCount: sort by totalStitches DESC
- stitchingDays: sort by stitchingDays DESC

Search: `useState<string>("")` for search text. Filter projects where `projectName.toLowerCase().includes(search.toLowerCase())`. Apply before sorting.

**Aggregate stat cards** (violet tint per UI-SPEC):
- Total shown: `projects.length`
- Total Stitches: sum of `totalStitches`
- Avg Duration: mean of `startToFinishDays` (non-null only)
- Avg Daily Stitches: mean of `avgDailyStitches`
- Grid: `grid grid-cols-[repeat(auto-fill,minmax(140px,1fr))] gap-3`
- Card styling: `bg-violet-50 border-violet-100` (distinct from emerald hero stats per UI-SPEC)
- Labels: `text-[11px] font-bold uppercase tracking-wider text-muted-foreground`
- Values: `font-mono text-lg font-bold tabular-nums`

Sort bar + search: `flex flex-wrap items-center justify-between gap-3` with search input on left, sort dropdown on right. Sort dropdown follows same pattern as WhatsNextTab (`<select>` element).

**Empty states** per UI-SPEC copywriting contract:
- No finished projects ever: "No finished projects yet. Your first finish is going to feel amazing!"
- Search returns nothing: "No finished projects match your search."
