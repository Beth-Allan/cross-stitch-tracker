**Shopping page.tsx** -- replace contents:
```typescript
import { getShoppingCartData } from "@/lib/actions/shopping-cart-actions";
import { getPresignedImageUrls } from "@/lib/actions/upload-actions";
import { ShoppingCart } from "@/components/features/shopping/shopping-cart";

export default async function ShoppingPage() {
  const data = await getShoppingCartData();

  // Collect cover thumbnail keys from projects
  const imageKeys = data.projects
    .map((p) => p.coverThumbnailUrl)
    .filter(Boolean) as string[];
  const imageUrls = await getPresignedImageUrls(imageKeys);

  return (
    <div className="space-y-6">
      <div>
        <h1 className="font-heading text-2xl font-semibold">Shopping Cart</h1>
        <p className="text-muted-foreground text-sm">
          Select projects to build your shopping list
        </p>
      </div>

      <ShoppingCart data={data} imageUrls={imageUrls} />
    </div>
  );
}
```

**TopBar modification** -- add custom event listener:

Find the TopBar component. It already has `useState` for `showLogModal`. Add a `useEffect` that listens for the custom "open-log-session-modal" event:

```typescript
// Add inside the TopBar component, alongside existing hooks:
useEffect(() => {
  function handleOpenLogModal() {
    setShowLogModal(true);
  }
  window.addEventListener("open-log-session-modal", handleOpenLogModal);
  return () => {
    window.removeEventListener("open-log-session-modal", handleOpenLogModal);
  };
}, []);
```

This is a minimal 7-line addition. No changes to existing TopBar behavior -- it just adds a new trigger source for the same modal. The existing "Log Stitches" button in TopBar continues to work as before.

Verify: The QuickAddMenu (from Plan 07) dispatches `window.dispatchEvent(new CustomEvent("open-log-session-modal"))` when "Log Stitches" is clicked. TopBar receives this event and opens the modal.
