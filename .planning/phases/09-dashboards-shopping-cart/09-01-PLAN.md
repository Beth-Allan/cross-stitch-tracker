Create `src/lib/actions/dashboard-actions.ts` with `"use server"` directive.

Import `requireAuth` from `@/lib/auth-guard`, `prisma` from `@/lib/db`, types from `@/types/dashboard`.

**getMainDashboardData()** — single exported function that calls `requireAuth()`, then `Promise.all()` over 5 internal helpers:

1. `getCurrentlyStitchingProjects(userId)`: Query `prisma.project.findMany` where `userId` and `status: { in: ["IN_PROGRESS", "ON_HOLD"] }`, include `chart` (select name, stitchCount, coverThumbnailUrl, designer.name), include `sessions` (select date, stitchCount, timeSpentMinutes, order by date desc). Map to `CurrentlyStitchingProject[]`:
   - `progressPercent = chart.stitchCount > 0 ? Math.min(100, Math.round((stitchesCompleted / chart.stitchCount) * 100)) : 0`
   - `lastSessionDate = sessions[0]?.date ?? null`
   - `totalTimeMinutes = sessions.reduce((sum, s) => sum + (s.timeSpentMinutes ?? 0), 0)`
   - `stitchingDays = new Set(sessions.map(s => s.date.toISOString().split("T")[0])).size`
   - Sort by `lastSessionDate` DESC, null dates last

2. `getStartNextProjects(userId)`: Query `prisma.chart.findMany` where project has `userId`, `wantToStartNext: true`, `status: { in: ["UNSTARTED", "KITTED"] }`, include designer, project, genres. Map to `StartNextProject[]`. Limit display to top 2 via `.slice(0, 2)`.

3. `getBuriedTreasures(userId)`: Query `prisma.chart.findMany` where project has `userId` and `status: "UNSTARTED"`, OR chart has no project (unstarted = no project record). Include designer, genres, project (select id). Sort by `dateAdded` ASC. Compute threshold: `Math.max(Math.ceil(allUnstarted.length * 0.1), 1)`. Slice to `Math.min(threshold, 5)`. Map to `BuriedTreasure[]` with `daysInLibrary = Math.floor((Date.now() - dateAdded.getTime()) / (1000*60*60*24))`.

4. `getCollectionStats(userId)`: Query all projects for user. Compute:
   - `totalProjects`: count all
   - `totalWIP`: count where status IN_PROGRESS
   - `totalOnHold`: count where status ON_HOLD
   - `totalUnstarted`: count where status UNSTARTED or KITTING or KITTED
   - `totalFinished`: count where status FINISHED or FFO
   - `totalStitchesCompleted`: sum of all project.stitchesCompleted
   - `mostRecentFinish`: project with latest finishDate (FINISHED or FFO), null if none
   - `largestProject`: project whose chart.stitchCount is highest, null if none

5. `getRandomSpotlightProject(userId)`: Query `prisma.project.count` for user. If 0, return null. Generate `skip = Math.floor(Math.random() * count)`. Query `prisma.project.findFirst` with skip, include chart (with designer, genres). Map to `SpotlightProject`.

**getSpotlightProject()** — separate export. Calls `requireAuth()`, then delegates to `getRandomSpotlightProject(user.id)`. This is the server action called by the Shuffle button.

All functions filter by `userId: user.id` on every query. Never use fallback user IDs.
