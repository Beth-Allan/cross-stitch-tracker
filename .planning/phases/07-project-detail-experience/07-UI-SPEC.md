## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, swatch-to-text gap in supply rows |
| sm | 8px | Compact element spacing, supply row internal padding |
| md | 16px | Default element spacing, card padding, tab content gap |
| lg | 24px | Section padding, hero metadata spacing |
| xl | 32px | Layout gaps between major sections (hero to tabs, tab content sections) |
| 2xl | 48px | Hero vertical padding (with cover image) |
| 3xl | 64px | Not used this phase |

Exceptions:
- **44px minimum touch targets** on: status badge dropdown trigger, kebab menu trigger, tab triggers, editable number click areas, supply row action buttons
- **12px** (`gap-3`) for tight inline metadata groups (stitch count + size badge + progress)

- All numeric values in supply rows and totals use `tabular-nums` for column alignment.

## Layout Contract

### Hero Section

```
+------------------------------------------------------------------+
| [blurred tinted bg fill]                                         |
|   +------------------------------------------+                   |
|   |  Cover Image (object-contain, max-h-64)  |                   |
|   +------------------------------------------+                   |
+------------------------------------------------------------------+
| Back to Gallery                                    [Edit] [...]  |
| Chart Name (Display, Fraunces, bold)                             |
| Designer Name (Body, muted-foreground)                           |
| [StatusBadge v] | 45,000 stitches | Large | 68%                 |
+------------------------------------------------------------------+
```

- **With cover image:** Full-width banner (max-h-64 / 256px). Image uses `object-contain` centered. Background fill is a CSS `backdrop-filter: blur(20px)` on a scaled-up copy of the image, or a tinted `bg-muted` fallback.
- **Without cover image:** Skip banner entirely. Compact hero with metadata only. Strong typography (Fraunces heading) + status accent color carries the visual weight. No placeholder image.
- **Hero metadata row:** Inline flex with `gap-3`. Items wrap on mobile.
- **Edit button:** Visible primary action in hero bar, right-aligned alongside kebab. Uses `LinkButton` with `variant="outline" size="sm"`.
- **Kebab menu:** `MoreHorizontal` icon trigger, 44px touch target. Contains: "Delete Project" (destructive).
- **Status badge:** Clickable. Shows current status + chevron-down indicator. Opens Select dropdown. 44px minimum touch target.

### Hero Responsive Behavior

- **Desktop (>= 1024px):** Banner at full width, metadata in single row, Edit + kebab right-aligned
- **Tablet (>= 768px):** Same layout, banner height reduces to max-h-48
- **Mobile (< 768px):** Banner reduces to max-h-40. Metadata stacks vertically. Edit + kebab in a row below chart name. Status badge + stats wrap.

### Tab Layout

- Two tabs: **Overview** and **Supplies**
- Use `variant="line"` (underline style, not pill)
- Active tab state stored in URL search param: `?tab=overview` (default) / `?tab=supplies`
- Tab triggers have 44px minimum height for touch targets
- Tab content has `pt-6` (24px) padding from tab bar

### Overview Tab (Status-Aware Sections)

Section ordering by status:

| Status | Section Order |
|--------|---------------|
| UNSTARTED | Kitting Checklist > Pattern Details > Dates > Project Setup |
| KITTING | Kitting Checklist > Pattern Details > Dates > Project Setup |
| KITTED | Pattern Details > Project Setup > Dates |
| IN_PROGRESS | Progress > Pattern Details > Dates > Project Setup |
| ON_HOLD | Progress > Pattern Details > Dates > Project Setup |
| FINISHED | Completion Stats > Pattern Details > Dates |
| FFO | Completion Stats > Pattern Details > Dates |

Each section uses `InfoCard` wrapper. Section heading uses `text-xl font-heading font-semibold`.

### Supplies Tab Layout

```
+------------------------------------------------------------------+
| Calculator Settings Bar (contextual -- hidden until first         |
| stitch count entered)                                            |
| Strands: [2]  |  Over: [2]  |  Fabric: [14ct]  |  Waste: [20%] |
+------------------------------------------------------------------+
| Threads (24 colours)                    12,450 stitches total    |
|   [supply row]                                                   |
|   [supply row]                                                   |
|   [+ Add thread colours]                                         |
|   ─────────────────────────────────────────────────────────────  |
|   Totals:  12,450 stitches | 48 skeins | 42 acquired            |
+------------------------------------------------------------------+
| Beads (3 colours)                                                |
|   [supply row]                                                   |
|   [+ Add beads]                                                  |
+------------------------------------------------------------------+
| Specialty Items (1 item)                                         |
|   [supply row]                                                   |
|   [+ Add specialty items]                                        |
+------------------------------------------------------------------+
```

### Supply Row Layout (Two-Line)

```
Line 1: [swatch 20x20] DMC 310 - Black                    [trash]
Line 2: 1,250 stitches -> 3 skeins | Need: [3] | Have: [2] | [partial]
```

- **Line 1:** Color swatch (20x20px rounded-sm, 1px border for light colors) + color code (mono, semibold) + color name (body). Trash icon right-aligned, 44px touch target, `text-muted-foreground hover:text-destructive`.
- **Line 2:** Stitch count (mono, tabular-nums) + arrow indicator + calculated skeins + divider + Need (EditableNumber) + Have (EditableNumber) + fulfillment status icon.
- **Fulfillment status:** Check icon (`text-success`) when have >= need. Warning icon (`text-warning`) when partial. No icon when have = 0.
- **Auto-calculated indicator:** When skeins are auto-calculated, show a small `Calculator` icon (12px, `text-muted-foreground`) next to the Need value. Tooltip: "Auto-calculated from stitch count".
- **Manual override indicator:** When need differs from calculated, show "Calc: X" in `text-xs text-warning-muted-foreground` below the Need value. Tooltip on hover: "Recalculated value is X skeins. Click to accept."
- **Beads/Specialty rows:** Same two-line layout. Line 2 shows: quantity needed + quantity acquired + fulfillment. No stitch count or calculation for these types (unless stitch count is entered).

### Supply Row Responsive Behavior

- **Desktop:** Two-line row as described above, full width
- **Mobile (< 768px):** Two-line row compresses. Line 2 numbers stack into a compact 2x2 grid: stitches/skeins on left, need/have on right. Fulfillment icon moves to line 1 (after color name).

### Calculator Settings Bar

- Appears as a compact toolbar at the top of the Supplies tab, styled with `bg-muted rounded-lg px-4 py-3`
- **Hidden by default** -- only appears when at least one supply row has a stitch count > 0 (progressive disclosure per D-17)
- Four inline-editable fields in a horizontal row:
  - **Strands:** Number input, default 2, range 1-6
  - **Over:** Toggle "1" or "2", default 2
  - **Fabric count:** Number showing linked fabric count or "14ct (default)". If no fabric linked, shows hint text.
  - **Waste factor:** Percentage, default 20%, range 0-50%
- Labels use 11px uppercase tracking-wider style (compact, informational)
- Each field uses inline editing (click to edit, Enter to save, Escape to cancel)
- On mobile: wraps to 2x2 grid

## Interaction Contract

### Status Change

1. User clicks status badge in hero (has chevron-down indicator)
2. Select dropdown opens with all 7 status options
3. User selects new status
4. Optimistic update: badge changes immediately
5. Server action fires, toast confirms
6. On failure: badge reverts, error toast
7. Overview tab sections reorder automatically based on new status

### Supply Quantity Editing

1. User clicks a Need or Have value (EditableNumber)
2. Field becomes an input, focused and selected
3. User types new value, presses Enter or clicks away
4. Server action fires optimistically
5. On failure: value reverts, error toast

### Stitch Count Entry

1. User clicks stitch count area on a supply row (or it's empty/0)
2. Inline input appears, focused
3. User enters stitch count, presses Enter
4. Skeins auto-calculate using formula: `ceil(stitches * strands / (fabricCount * overCount * stitchesPerSkein) * (1 + wasteFactor))`
5. Need value updates with calculator icon indicator
6. If Need was previously manually overridden: show "Calc: X" warning below Need. User decides.

### Manual Override Flow

1. User edits the Need value on a row that was auto-calculated
2. Calculator icon changes to manual override state
3. If stitch count later changes: new calculation shown as "Calc: X" suggestion
4. Override is never silently overwritten (per D-13)

### Supply Addition (SearchToAdd Enhanced)

1. User clicks "+ Add thread colours" (or beads/specialty)
2. SearchToAdd panel opens with search input
3. User types to filter catalog
4. **If match found:** User clicks item. Row added with stitch count = 0, need = 1.
5. **If no match:** Bottom of list shows "+ Create [search text]". User clicks it.
6. InlineSupplyCreate form appears (name, optional color code/hex for threads). Minimal fields.
7. After creation: item added to catalog AND to project supply list.
8. SearchToAdd stays open for multi-add (existing behavior).

### Supply Deletion

1. User clicks trash icon on supply row
2. Confirm with toast or immediate deletion (low-stakes -- single row, not project)
3. Row removed optimistically
4. On failure: row reappears, error toast

### Project Deletion (from Kebab)

1. User clicks kebab menu > "Delete Project"
2. Confirmation dialog opens: "Delete [chart name]?"
3. Body: "This will permanently delete this project and all its supplies. This cannot be undone."
4. Two buttons: "Cancel" (outline) and "Delete Project" (destructive)
5. On confirm: redirect to gallery, success toast
6. On failure: dialog stays open with error message (retry-able)

### Tab Switching

1. User clicks tab trigger (Overview or Supplies)
2. URL param updates (`?tab=overview` or `?tab=supplies`)
3. Tab content swaps (no page reload)
4. Deep links work: navigating to `/charts/[id]?tab=supplies` opens supplies tab directly

### Settings Bar Interactions

1. Settings bar is hidden when no supply row has stitch count > 0
2. When first stitch count is entered on any row, settings bar slides in (animated)
3. User can click any setting value to edit inline
4. Changes apply immediately to all calculated supply rows
5. Recalculated values that differ from manual overrides show "Calc: X" warnings

## Accessibility Contract

| Element | Requirement |
|---------|-------------|
| Status badge dropdown | `role="combobox"` via shadcn Select; `aria-expanded`, `aria-label="Change project status"` |
| Kebab menu | `aria-label="Project actions"` on trigger button |
| Tabs | Base UI Tabs provides `role="tablist"`, `role="tab"`, `role="tabpanel"` automatically |
| Editable numbers | `aria-label="[field name] for [color name]"` (e.g., "Quantity needed for DMC 310") |
| Supply rows | `role="list"` on section container, `role="listitem"` on each row |
| Progress bar | `role="progressbar"` with `aria-valuenow`, `aria-valuemin`, `aria-valuemax` (existing) |
| Calculator icon | `aria-label="Auto-calculated"` on indicator |
| Override warning | Live region or tooltip (not hidden from screen readers) |
| Delete dialog | Focus trapped, ESC to close (existing Dialog behavior) |
| All buttons | 44px minimum touch target |
| Color swatches | Decorative (`aria-hidden="true"`) since color code + name provide text alternative |
