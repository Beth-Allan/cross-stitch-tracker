| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), sort chevron padding |
| sm | 8px | Table cell vertical padding (`py-2`), compact inline spacing |
| md | 16px | Table cell horizontal padding (`px-4`), form field gap, card padding |
| lg | 24px | Page content padding (`px-6 pt-6 pb-4`), section spacing (`space-y-6`) |
| xl | 32px | Header-to-content gap, modal inner padding (`px-8`) |

## Color

All colors use semantic tokens from `globals.css`. Never use raw color scales (stone-*, emerald-*).

| Role | Token | Light Value | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `bg-background` | stone-50 | Page background, main content area |
| Secondary (30%) | `bg-card` / `bg-muted` | white / stone-100 | Table container, card surfaces, modal background, mobile cards |
| Primary accent (10%) | `bg-primary` / `text-primary` | emerald-600 | "Add Designer" / "Add Genre" CTA button, active sort column text, website links, edit button hover, focus rings |
| Destructive | `text-destructive` / `bg-destructive` | red-600 | Delete button hover, delete confirmation button |
| Genre tag | amber-50 bg + amber-700 text + amber-200 border | amber palette | Top genre badge on designer rows and detail pages |
| Muted text | `text-muted-foreground` | stone-500 | Table header labels, helper text, placeholder text, stat labels, empty dash |
| Border | `border-border` | stone-300 | Table row dividers, card borders, input borders, modal separator |
| Ring / focus | `ring` | emerald-600 | Focus-visible ring on inputs and buttons |

## Interaction Patterns

### Sortable Table Headers

- **Default sort:** Name ascending (A-Z) for both designer and genre tables
- **Click behavior:** First click sorts ascending; second click on same column toggles to descending; clicking a different column sorts ascending
- **Visual indicator:** Active column header text uses `text-primary` (emerald). Inactive headers use `text-muted-foreground`. ChevronUp for ascending, ChevronDown for descending (12px, `w-3 h-3`)
- **Sortable columns (designers):** Designer name, Charts count, Finished count
- **Sortable columns (genres):** Genre name, Charts count
- **Non-sortable columns (designers):** Web, Started, Top Genre

### Search

- **Placeholder:** "Search designers..." / "Search genres..."
- **Behavior:** Client-side text filter on name field, instant filtering (no debounce needed for small datasets)
- **Icon:** Search icon (16px) positioned absolute left inside input
- **Clear:** "Clear all" text button appears when any filter or search is active

### Row Actions (Desktop)

- **Hover reveal:** Edit (Pencil) and Delete (Trash2) icons appear on row hover, 14px icons (`w-3.5 h-3.5`)
- **Edit icon:** `text-muted-foreground` default, `hover:text-foreground hover:bg-muted` on hover
- **Delete icon:** `text-muted-foreground` default, `hover:text-destructive hover:bg-destructive/10` on hover
- **Row click (designer name):** Navigates to detail page (`/designers/[id]`)

### Row Actions (Mobile Cards)

- **Always visible:** Edit and Delete icons shown in card header, not hidden behind hover
- **Card tap:** Navigates to detail page

### Modal Forms

- **Open triggers:** "Add Designer/Genre" button on list page; Edit button on detail page
- **Close:** X button, backdrop click, or Cancel button
- **Form reset:** On open, populate with entity data (edit mode) or clear fields (create mode)
- **Validation:** Name required with trim, shown inline below field. Duplicate name shows server error.
- **Submit:** Primary button shows "Adding..." / "Saving..." during pending state (useTransition)
- **Success:** Toast notification via sonner ("Designer created" / "Designer updated"), dialog closes, list refreshes via revalidatePath

### Delete Confirmation

- **Trigger:** Delete icon on row (list page) or Trash2 button on detail page
- **Dialog:** Shows entity name, affected chart count, and reassurance that charts will NOT be deleted
- **Confirm button:** `variant="destructive"`, shows "Deleting..." during pending state
- **Success (from list):** Toast "Designer deleted", list refreshes
- **Success (from detail):** Toast "Designer deleted", redirects to list page via `router.push`

### Chart List on Detail Pages

- **Sort pills:** Inline text buttons right-aligned: Name, Stitches, Status
- **Active pill:** `bg-emerald-50 text-emerald-700` with sort direction chevron
- **Chart row:** Clickable, navigates to `/charts/[id]`
- **Cover thumbnail:** 40x40px (`w-10 h-10`) rounded-lg. Placeholder uses Image icon from lucide if no cover
- **Chart info:** Name (font-medium), stitch count + size category below (text-xs text-muted-foreground)
- **Status:** StatusBadge component on the right. Progress bar shown for In Progress charts only.

### Navigation

- **Sidebar items:** "Designers" (Paintbrush icon) and "Genres" (Tags icon) inserted after "Charts" in nav-items.ts
- **Back navigation:** ArrowLeft icon + "Back to Designers" / "Back to Genres" link at top of detail pages, matching chart detail page pattern

## Responsive Breakpoints

| Breakpoint | Designer List | Genre List | Detail Pages |
|------------|---------------|------------|--------------|
| < md (mobile) | Card layout, stacked stats | Card layout | Stats stack vertically, chart cards full width |
| >= md (tablet+) | Full table with all columns | Full table (2 columns) | Stats row horizontal, chart cards full width |

**Source:** DesignOS `DesignerPage.tsx` uses `max-md:hidden` for table, `md:hidden` for cards. Charts page uses `hidden sm:table-cell`, `hidden md:table-cell`, `hidden lg:table-cell` for progressive column display.

**Phase 3 approach:** Follow DesignOS pattern -- full table at `md:`, card layout below `md:`. Designer table has fewer columns than charts, so no need for intermediate breakpoints.

## Loading & Error States

### Loading States

| Page | Loading Pattern |
|------|----------------|
| Designer list | Server Component renders with data; no client-side loading needed |
| Designer detail | Server Component renders with data; `notFound()` if ID doesn't exist |
| Genre list | Same as designer list |
| Genre detail | Same as designer detail |
| Modal form submission | Button text changes to "Adding..." / "Saving..." / "Deleting...", button disabled |

### Error States

| Error | Handling |
|-------|----------|
| Designer/genre not found (detail page) | Next.js `notFound()` renders 404 page |
| Server action failure | Toast error: "Something went wrong. Please try again." |
| Duplicate name on create/edit | Inline error below name field: "A designer with that name already exists" |
| Validation error (Zod) | Inline error below the relevant field |
| Network error during mutation | try/catch in client, toast error message |
