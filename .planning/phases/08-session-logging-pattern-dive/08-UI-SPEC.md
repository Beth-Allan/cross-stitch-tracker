## Typography

4 sizes, 2 weights. All roles mapped to one of these.

| Role | Size | Weight | Line Height | Font Family | Usage |
|------|------|--------|-------------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Source Sans 3 | Table cells, descriptions, session counts, hint text, count summaries, sub-labels |
| Micro | 12px | 600 (semibold) | 1.4 | Source Sans 3 | Uppercase tracking-wider labels (DATE, STITCHES, TOTAL STITCHES), section headers, fabric count headers |
| Heading | 18px | 600 (semibold) | 1.3 | Fraunces | Page subtitle, project names in cards |
| Display | 24px | 600 (semibold) | 1.2 | Fraunces | "Pattern Dive" page title |

**Stat values** use 18px / 600 (semibold) in JetBrains Mono with `font-variant-numeric: tabular-nums`. Same size and weight as Heading role but different font family for numeric alignment.

Notes:
- All uppercase labels use `tracking-wider` (letter-spacing: 0.05em)
- Stat values use `font-variant-numeric: tabular-nums` for alignment
- Weights used in this phase: 400 (regular) and 600 (semibold) only
- Previous micro labels at 10-11px consolidated to 12px for legibility and scale compliance
- Previous helper text at 12-13px consolidated to 14px (Body role)

## Interaction Patterns

### LogSessionModal

| Interaction | Behavior |
|-------------|----------|
| Open from TopBar | Full modal with project picker shown, date defaults to today |
| Open from project detail | Modal pre-fills project, project picker hidden (locked) |
| Open from sessions page | Full modal with project picker shown |
| Project picker | Custom searchable dropdown with mini cover images (28px thumbnails) |
| Project search | Filters active projects by name, instant client-side filtering |
| Stitch count validation | Required, integer, min 1 |
| Time spent | Optional, split into hours + minutes fields |
| Photo upload | Optional, dashed border button, reuses R2 presigned URL pattern |
| Edit mode | Same modal, pre-populated fields, "Delete session" link appears in footer |
| Delete flow | Two-step inline confirmation: "Delete session" text -> "Delete? [Yes] [No]" buttons |
| Save | Disabled state when project or stitch count empty (opacity-40) |
| Close | Backdrop click or dismiss button (see Copywriting Contract for dismiss label) |

### Session Table (ProjectSessionsTab & SessionsPage)

| Interaction | Behavior |
|-------------|----------|
| Sort by column | Click header to toggle: date (default desc), stitches, time. Active column shows emerald arrow icon |
| Edit session | Pencil icon appears on row hover (opacity-0 -> opacity-100 transition). Opens LogSessionModal in edit mode |
| Photo indicator | Camera icon in emerald when session has a photo |
| Empty state | Centered text, no table rendered |

### Pattern Dive Tabs

| Interaction | Behavior |
|-------------|----------|
| Tab switching | Instant, no loading spinners. All data fetched eagerly via Promise.all() in server component |
| URL persistence | nuqs `?tab=browse` (default), `?tab=whats-next`, `?tab=fabric`, `?tab=storage` |
| Tab icons | Each tab has a lucide icon: Search (Browse), Star (What's Next), Layers (Fabric Requirements), MapPin (Storage View) |
| Active tab style | Emerald bottom border, emerald text. Inactive: transparent border, stone-500 text |

### What's Next Tab

| Interaction | Behavior |
|-------------|----------|
| Sort dropdown | Select element with options: Kitting Readiness (default), Oldest First, Newest First, Largest First, Smallest First |
| Card click | Navigate to project detail page |
| Card hover | Subtle shadow + border color change |
| Kitting bar | Emerald at 100%, amber below 100%. Text label: "Fully kitted" or "Kitting" |
| Priority badge | Star icon + number for wantToStartNext projects, em-dash for others |

### Fabric Requirements Tab

| Interaction | Behavior |
|-------------|----------|
| Filter toggle | Two pill buttons: "Needs Fabric" (default active), "All Projects" |
| Expand/collapse | Click row to expand detailed fabric matching section |
| Size reference | Collapsible sub-section within expanded card, chevron rotates on open |
| Assign fabric | "Assign" pill button on matching fabric row, becomes "Assigned" badge after click |
| Status icons | Check (emerald) = fabric assigned and fits, AlertTriangle (amber) = fabric too small, Package (stone) = no fabric |

### Storage View Tab

| Interaction | Behavior |
|-------------|----------|
| Collapse/expand groups | Click location header to toggle. Default: all expanded |
| Item click | Navigate to project detail or fabric detail |
| Thumbnails | 36px rounded squares. Projects show cover image or status-gradient placeholder. Fabrics show Layers icon |

### Progress Auto-Update (D-04, D-05, D-06)

| Interaction | Behavior |
|-------------|----------|
| After session create/edit/delete | stitchesCompleted recalculated atomically ($transaction) |
| Progress display (sessions exist) | Read-only display, no EditableNumber |
| Progress display (no sessions) | EditableNumber behavior preserved |
| One-time migration | Existing stitchesCompleted copied to startingStitches for projects with progress but no sessions |

## State Specifications

### Loading States
- Pattern Dive page: Server-rendered, no client loading states. All tab data fetched via `Promise.all()` before render
- LogSessionModal: No loading state on open (project list passed as props). Save button should show pending state during server action

### Error States
- Session save failure (validation): Toast notification "Session could not be saved. Check that stitch count is a positive number and a project is selected." with rollback of any optimistic updates
- Session save failure (network): Toast notification "Session could not be saved. Check your connection and try again." with rollback of any optimistic updates
- Session delete failure: Toast notification "Session could not be deleted. Check your connection and try again."
- Photo upload failure: Toast notification "Photo upload failed. Your session was saved without the photo."

### Optimistic Updates
- Session create: Append to session list immediately, recalculate stats. Rollback on server error
- Session edit: Update row immediately. Rollback on server error
- Session delete: Remove row immediately, recalculate stats. Rollback on server error

## Mini-Stat Card Specification

The sessions tab uses a 4-column grid of mini-stat cards. Each card follows this exact layout:

```
[icon 13px] [UPPERCASE LABEL 12px tracking-wider semibold]
[VALUE 18px semibold tabular-nums]
```

| Stat | Label | Icon | Mono font? |
|------|-------|------|-----------|
| Total Stitches | TOTAL STITCHES | Activity (13px) | Yes (JetBrains Mono) |
| Sessions Logged | SESSIONS LOGGED | Hash (13px) | Yes |
| Avg per Session | AVG PER SESSION | TrendingUp (13px) | Yes |
| Active Since | ACTIVE SINCE | Calendar (13px) | No (Source Sans 3, it's a date) |

Card background: `bg-stone-50 dark:bg-stone-800/50`
Card border-radius: `rounded-lg` (8px)
Card padding: 12px top, 16px sides
Grid gap: 12px
Responsive: 4 columns on desktop, 2 columns on mobile
