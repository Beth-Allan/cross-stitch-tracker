**3. Run one-time data migration for D-05:**

Execute the migration SQL to copy stitchesCompleted to startingStitches for projects with progress but no sessions:

```bash
npx prisma db execute --stdin <<'SQL'
UPDATE "Project"
SET "startingStitches" = "stitchesCompleted"
WHERE "stitchesCompleted" > 0
  AND "startingStitches" = 0;
SQL
```

This migration is idempotent (the WHERE clause ensures it only affects rows where startingStitches hasn't been set). It's safe to run multiple times. Per pitfall 3 from RESEARCH.md, the condition excludes projects where startingStitches was already manually set.

Note: We omit the `NOT EXISTS (SELECT 1 FROM "StitchSession" WHERE ...)` clause from the pitfall because this migration runs before any sessions exist in the database. The simpler condition is sufficient and equivalent.

**4. Verify migration:**
```bash
npx prisma db execute --stdin <<'SQL'
SELECT COUNT(*) as migrated_count
FROM "Project"
WHERE "startingStitches" > 0 AND "startingStitches" = "stitchesCompleted";
SQL
```

This shows how many projects were migrated.
