<objective>
Fix the Catch-22 in `getFabricRequirements()` where fabric matching always returns empty for unassigned projects.

Purpose: The "Needs Fabric" view in Pattern Dive's Fabric Requirements tab shows projects without assigned fabric, but the matching logic derives `fabricCount` from the assigned fabric -- which is always null for these projects. This makes `matchingFabrics` always `[]`, so users see "No fabrics in your stash fit this project" even when they have fitting fabrics. UAT Test 13 failed because of this.

Output: Fixed `getFabricRequirements()` that computes matching per-fabric (using each unassigned fabric's own count) for projects without assigned fabric. No type changes needed -- `FabricRequirementRow.matchingFabrics` shape is unchanged.
</objective>

Replace the current matching block:

```typescript
const fabricCount = p.fabric?.count ?? null;
// ... requiredWidth/requiredHeight from fabricCount ...
// ... matchingFabrics filtered by f.count === fabricCount ...
```

With dual-path logic:

**Path A -- Project HAS assigned fabric (existing behavior, preserved):**
- `fabricCount` = assigned fabric's count
- `requiredWidth` = `stitchesWide / fabricCount + MARGIN_TOTAL` (rounded)
- `requiredHeight` = `stitchesHigh / fabricCount + MARGIN_TOTAL` (rounded)
- `matchingFabrics` = unassigned fabrics filtered by `f.count === fabricCount`, then check size fits

**Path B -- Project has NO assigned fabric (new behavior):**
- `fabricCount` = null (unchanged)
- `requiredWidth` = null (unchanged -- no single "required" size without a count)
- `requiredHeight` = null (unchanged)
- `matchingFabrics` = iterate ALL unassigned fabrics, for EACH fabric:
  - Compute `reqW = Math.round((stitchesWide / f.count + MARGIN_TOTAL) * 10) / 10`
  - Compute `reqH = Math.round((stitchesHigh / f.count + MARGIN_TOTAL) * 10) / 10`
  - Check `fitsWidth`: `f.shortestEdgeInches >= reqW || f.longestEdgeInches >= reqW`
  - Check `fitsHeight`: `f.shortestEdgeInches >= reqH || f.longestEdgeInches >= reqH`
  - Include the fabric in matchingFabrics if `fitsWidth || fitsHeight` (show partial fits too -- the UI already renders fitsWidth/fitsHeight flags)
  - Shape is identical to existing: `{ id, name, brandName, count, shortestEdgeInches, longestEdgeInches, fitsWidth, fitsHeight }`

This approach:
- Removes the Catch-22: no dependency on assigned fabric for matching
- Preserves the FabricRequirementRow type unchanged
- Preserves the UI component unchanged (it already handles matchingFabrics correctly)
- Preserves existing behavior for assigned-fabric projects
