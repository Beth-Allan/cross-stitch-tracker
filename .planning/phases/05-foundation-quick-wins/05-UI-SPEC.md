## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon gaps, inline padding, icon button padding |
| sm | 8px (`gap-2`, `p-2`) | Compact element spacing, inline edit input padding |
| md | 16px (`gap-4`, `p-4`) | Default element spacing, list row padding, form field gaps |
| lg | 24px (`gap-6`, `p-6`) | Section padding, page-level `px-6 pt-6 pb-4` from design |
| xl | 32px (`gap-8`, `p-8`) | Layout gaps between major sections |
| 2xl | 48px (`py-12`) | Not expected in Phase 5 |
| 3xl | 64px (`py-16`) | Empty state vertical padding only |

Exceptions:
- Touch target for inline edit confirm/cancel buttons: `p-1.5` (6px) per DesignOS design. Acceptable deviation for icon-only controls that cluster together.
- List row padding: `px-4 py-3` (16px/12px) per DesignOS StorageLocationList design.

**Source:** DesignOS `StorageLocationList.tsx` and `StorageLocationDetail.tsx` spacing values.

## Interaction Contracts

### Inline Name Edit

| Trigger | Behavior |
|---------|----------|
| Click pencil icon on list row | Row switches to edit mode: input replaces name text, auto-focus + auto-select |
| Click pencil icon on detail header | Header switches to edit mode with larger input (text-2xl, font-heading) |
| Enter key (if trimmed value non-empty) | Saves name via server action, exits edit mode |
| Escape key | Cancels edit, restores original name |
| Blur event | Cancels edit (same as Escape) |
| Confirm button (checkmark) | Uses `onMouseDown` (not `onClick`) to fire before blur. Saves name. |
| Cancel button (X) | Uses `onMouseDown` to fire before blur. Cancels edit. |

**Why onMouseDown:** Clicking a button triggers blur on the input first. If we use onClick, the blur handler fires first and cancels the edit before the save can execute. onMouseDown fires before blur.

### Inline Add (Storage/App)

| Trigger | Behavior |
|---------|----------|
| Click "Add Location" / "Add App" CTA | Inserts inline add row at the top of the list with emerald-tinted background |
| Enter key (if trimmed value non-empty) | Creates entity via server action, closes inline add row |
| Escape key | Closes inline add row |
| Blur event | Closes inline add row |
| "Add" button | Uses `onMouseDown`. Creates entity. Disabled when input empty (opacity-40, cursor-not-allowed). |

**Placeholder text:**
- Storage: "Location name, e.g. Bin A, Bookshelf 2..."
- Apps: "App name, e.g. Markup R-XP, Pattern Keeper..."

### Delete Confirmation Dialog

| Trigger | Behavior |
|---------|----------|
| Click trash icon on list row or detail page | Opens `DeleteEntityDialog` with entity name and affected project count |
| "Cancel" button | Closes dialog |
| "Delete" button | Calls delete server action (shows "Deleting..." while pending), closes on success |
| Dialog describes FK unlink | Copy includes: "{N} project(s) will be unlinked from this {entity type}. Projects will NOT be deleted." |

**Entity types for delete dialog:** Extend existing `entityType` union from `"designer" | "genre" | "brand" | "supply"` to include `"storage-location" | "stitching-app"`.

### List Row Hover

| State | Visual |
|-------|--------|
| Default | Pencil, trash, chevron icons hidden (`opacity-0`) |
| Hover | Icons fade in (`opacity-100`), row border darkens, shadow elevates (`shadow-sm` to `shadow-md`) |
| Row click | Navigates to detail page (`/storage/[id]` or `/apps/[id]`) |
| Pencil click | `e.stopPropagation()` -- enters edit mode without navigating |
| Trash click | `e.stopPropagation()` -- opens delete dialog without navigating |

### Fabric Selector (SearchableSelect)

| State | Visual |
|-------|--------|
| Unassigned fabrics exist | Standard dropdown with options formatted: `"{name} - {count}ct {type} ({brand})"` |
| No unassigned fabrics | Dropdown disabled, placeholder "No unassigned fabrics", helper text below: `Add fabric` (link to /fabric) `to assign to this project` |
| Currently linked fabric (edit mode) | Shows in dropdown even though its `linkedProjectId` is set (it belongs to this project) |

### Cover Image Display

| State | Before (current) | After (Phase 5) |
|-------|-------------------|------------------|
| Container height | `h-32` (128px) | `h-48` (192px) |
| Object fit | `object-cover` (crops) | `object-contain` (letterboxes) |
| Letterbox fill | none | `bg-muted` (stone-100 light / stone-800 dark) |
| Affects | Upload preview, resolving state | Upload preview, resolving state, drop zone, chart detail page |
| Does NOT affect | `cover-thumbnail.tsx` 10x10 thumbnails (keep `object-cover`) | same |

### Thread Picker Scroll

| Trigger | Behavior |
|---------|----------|
| User selects a thread to add | After successful add + toast, scroll the search/add controls into view |
| Scroll method | `ref.current?.scrollIntoView({ behavior: "smooth", block: "end" })` |
| Timing | `setTimeout(() => ..., 100)` to allow React to re-render the updated list |

## Semantic Token Translation

The DesignOS design files use hardcoded color scales (e.g. `bg-emerald-50`, `text-stone-500`). When implementing, translate to semantic tokens per `.claude/rules/base-ui-patterns.md`:

| DesignOS (hardcoded) | Implementation (semantic) |
|----------------------|---------------------------|
| `bg-white dark:bg-stone-900` | `bg-card` |
| `text-stone-900 dark:text-stone-100` | `text-foreground` or `text-card-foreground` |
| `text-stone-500 dark:text-stone-400` | `text-muted-foreground` |
| `text-stone-400 dark:text-stone-500` | `text-muted-foreground` (lighter weight context) |
| `border-stone-200 dark:border-stone-800` | `border-border` |
| `bg-stone-100 dark:bg-stone-800` | `bg-muted` |
| `bg-emerald-600 text-white` (CTA) | `bg-primary text-primary-foreground` |
| `text-emerald-600 dark:text-emerald-400` | `text-primary` |
| `bg-emerald-50/50 dark:bg-emerald-950/20` | `bg-primary/5` (approximate -- match the tint intent) |
| `border-emerald-200 dark:border-emerald-800` | `border-primary/20` |
| `ring-emerald-500/40` | `ring-primary/40` or `ring-ring/40` |
| `hover:text-red-500 dark:hover:text-red-400` | `hover:text-destructive` |
| `hover:bg-red-50 dark:hover:bg-red-950/30` | `hover:bg-destructive/10` |
| `text-stone-300 dark:text-stone-600` | `text-border` (chevron color -- muted border tone) |

**Important:** The inline add row's emerald tint (`bg-primary/5 border-primary/20`) is an intentional accent usage. This is the only place where a translucent primary background appears on a list item.
