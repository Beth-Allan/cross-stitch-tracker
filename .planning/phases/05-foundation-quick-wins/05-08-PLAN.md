  </behavior>
  <action>
    **search-to-add.tsx changes:**

    1. Add a `flipUp` state: `const [flipUp, setFlipUp] = useState(false);`

    2. Add a useEffect (not useLayoutEffect — jsdom doesn't support it and the visual flicker is negligible since the picker is absolutely positioned) that runs on mount to measure available space:

       ```typescript
       useEffect(() => {
         if (!ref.current) return;
         const rect = ref.current.parentElement?.getBoundingClientRect();
         if (!rect) return;
         const spaceBelow = window.innerHeight - rect.bottom;
         // max-h-72 = 288px, plus padding/border ~12px
         setFlipUp(spaceBelow < 300);
       }, []);
       ```

       Note: Use `ref.current.parentElement` because the picker is absolutely positioned relative to its parent. The parent's bottom edge tells us where the picker will render.

    3. Update the container div className on line 202. Change from:
       ```
       className="border-border bg-card absolute top-full right-0 left-0 z-20 mt-1 rounded-lg border shadow-lg"
       ```
       to:
       ```
       className={cn(
         "border-border bg-card absolute right-0 left-0 z-20 rounded-lg border shadow-lg",
         flipUp ? "bottom-full mb-1" : "top-full mt-1"
       )}
       ```

       Import `cn` from `@/lib/utils` (add to existing imports at top of file).

    **search-to-add.test.tsx changes:**

    Add a new describe block "SearchToAdd - viewport flip" with two tests:

    1. "uses top-full positioning when space below is sufficient":

       - Assert it does NOT have `top-full`
