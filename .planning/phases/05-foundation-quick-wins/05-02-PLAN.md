    **InlineNameEdit component (src/components/features/storage/inline-name-edit.tsx):**

    "use client" directive. Props interface:

    ```typescript
    interface InlineNameEditProps {
      name: string;
      onSave: (newName: string) => Promise<void>;
      variant?: "default" | "heading";
    }
    ```

    State: `isEditing` (boolean), `editValue` (string), `isPending` (from useTransition).

    Display mode: Show name text with appropriate classes based on variant:
    - `variant="default"`: `text-sm font-medium` (list row)
    - `variant="heading"`: `text-2xl font-heading font-semibold` (detail page)

    Edit mode: Show `<input>` with `ref` for auto-focus, auto-select all text on mount via `useEffect`. Classes based on variant:
    - `variant="default"`: `text-sm h-7 px-2` with `ring-primary/40`
    - `variant="heading"`: `text-2xl font-heading h-10 px-2` with `ring-primary/40`

    Key handlers:
    - Enter: if `editValue.trim().length > 0`, call `onSave(editValue.trim())` inside startTransition, then exit edit mode
    - Escape: reset editValue to original name, exit edit mode

    Blur handler: cancel edit (same as Escape).

    Confirm button (`Check` icon, `w-3.5 h-3.5`): `onMouseDown` (NOT onClick) calls save. `aria-label="Save name"`.
    Cancel button (`X` icon, `w-3.5 h-3.5`): `onMouseDown` cancels. `aria-label="Cancel editing"`.

    Per UI-SPEC: onMouseDown fires before blur, preventing the blur handler from canceling the save.
