## Spacing Scale

Declared values (multiples of 4, already in Tailwind v4 defaults):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Icon gaps (gap-1), inline padding, dot-to-label in badges |
| sm | 8px | Form field internal padding, compact gaps between badges |
| md | 16px | Default element spacing, form field vertical rhythm (space-y-4) |
| lg | 20px | Section padding inside cards (px-5 py-4 per design reference) |
| xl | 24px | Section gaps, grid gaps between info cards |
| 2xl | 32px | Page horizontal padding (p-5 lg:p-8 per design reference) |
| 3xl | 48px | Not used in this phase |

Exceptions:
- Form section dividers use pt-5 (20px) top padding with a top border, matching the design reference SectionHeading component
- Cover image upload drop zone: 128px min-height (h-32)
- Detail page cover image: max-height 320px (max-h-80) for hero display
- Touch targets: all interactive elements minimum 44px tap target on mobile

## Typography

| Role | Size | Weight | Line Height | Font Family | Usage |
|------|------|--------|-------------|-------------|-------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 (leading-normal) | Source Sans 3 | Form inputs, detail row values, general text |
| Label | 11px (text-xs) | 600 (semibold) | 1.5 | Source Sans 3 | Form field labels, section headings, detail row labels -- ALL CAPS with tracking-wider |
| Heading | 24px (text-2xl) | 600 (semibold) | 1.2 (leading-tight) | Fraunces | Page titles ("Add New Chart", chart name on detail page) |
| Stat | 14px (text-sm) | 500 (medium) | 1.5 | JetBrains Mono | Progress percentage, stitch count numbers on detail page ONLY |

Notes:
- Form field labels use uppercase tracking-wider pattern: `text-xs font-semibold uppercase tracking-wider text-stone-400 dark:text-stone-500`

- Info card titles on detail page use Fraunces at text-sm font-semibold
- Badge text: text-xs (12px) for status badges (sm), text-sm (14px) for status badges (md)
- Size badge: 10px uppercase tracking-wider font-semibold text-stone-400

## Interaction Patterns

### Form Submission
- Client-side validation with Zod before Server Action call
- Disable submit button during submission (loading state)
- Success: redirect to detail page with sonner toast "Chart added" / "Changes saved"
- Validation error: highlight fields with red border, show error message below field in text-xs text-destructive
- Server error: sonner error toast with message

### File Upload (Cover Photo)
- Drop zone with dashed border (border-dashed border-2 border-stone-200)
- Hover state: border-emerald-500 bg-emerald-50/50
- States: idle -> uploading (progress indicator) -> complete (show thumbnail) -> error (retry option)
- Upload indicator: replace drop zone content with "Uploading..." and a spinner or progress bar
- Complete: show uploaded image as thumbnail within the zone, with "Remove" overlay button
- Error: show error message within the zone with "Try again" link
- Accept: PNG, JPG, WEBP up to 5MB

### File Upload (Digital Working Copy)
- Compact upload button (not a drop zone)
- States: no file -> uploading -> uploaded (show filename + download link)
- Accept: PDF, PNG, JPG up to 5MB

### Status Change (Detail Page)
- Dedicated dropdown control (not buried in edit form, per D-16)
- Selecting a new status triggers Server Action immediately
- Sonner toast confirms: "Status changed to {label}"
- No confirmation dialog for status changes (non-destructive, reversible)

### Delete Chart
- "Delete" button on detail page triggers confirmation dialog
- Dialog requires explicit confirmation before deletion
- On confirm: Server Action deletes chart + project (cascade), redirect to /charts with toast "Chart deleted"

### Inline Entity Creation (Designer/Genre)
- Searchable select shows "Add {entity}" option at bottom of dropdown
- Clicking opens a Dialog with a single text input + Cancel/Add buttons
- On save: Server Action creates entity, dialog closes, new entity auto-selected in the form
- The form is NOT lost -- inline creation is non-destructive to form state

### Navigation
- Back link uses `<- Charts` pattern (ArrowLeft icon + "Charts" text)
- Back link hover: icon shifts left 0.5px (translate-x), text darkens
- Breadcrumb-style, not browser back -- always navigates to /charts

## States Inventory

Every view must handle these states:

### Charts List Page
| State | What renders |
|-------|-------------|
| Loading | Skeleton cards/rows (3-5 placeholder items) |
| Empty | Empty state with heading, body, and CTA button |
| Populated | List/table of charts |
| Error | Error message with retry |

### Chart Detail Page
| State | What renders |
|-------|-------------|
| Loading | Skeleton layout matching the detail structure |
| Not found | "Chart not found" with back link to /charts |
| Populated | Full detail view |
| Deleting | Delete dialog open, delete button in loading state |

### Chart Form (Add/Edit)
| State | What renders |
|-------|-------------|
| Idle | Empty form (add) or pre-populated form (edit) |
| Submitting | Submit button disabled with loading state, form fields disabled |
| Validation error | Red borders on invalid fields, error messages below fields |
| Server error | Sonner error toast, form returns to idle state |

### File Upload
| State | What renders |
|-------|-------------|
| Idle | Drop zone (cover) or upload button (working copy) |
| Dragging over | Highlighted drop zone border (emerald) |
| Uploading | Progress indicator or spinner |
| Complete | Thumbnail (cover) or filename + download link (working copy) |
| Error | Error message with retry option |
| R2 not configured | Disabled zone with "Storage not configured" message |

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| Mobile (<640px) | Single column layout. Form fills full width with p-5. Detail page info cards stack vertically. Date inputs stack (grid-cols-1). Bin/App selects stack. |
| Tablet (640-1024px) | Form at max-w-2xl centered. Detail page info cards start at 2-column grid. |
| Desktop (>1024px) | Form at max-w-2xl centered with p-8. Detail page uses full 2-column grid for info cards. |

Form footer (Cancel + Submit) always right-aligned, sticky on mobile if needed for long form scrolling.

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Focus visible | focus:ring-2 focus:ring-emerald-500/40 on all interactive elements (established in Phase 1) |
| Status badges | Colored dot is decorative, text label carries the meaning (color never sole indicator) |
| File upload | Drop zone has role="button" and keyboard activation (Enter/Space) |
| Form errors | aria-invalid on errored fields, aria-describedby linking to error message |
| Dialog | Focus trapped inside dialog, ESC closes, focus returns to trigger |
| Delete confirmation | Destructive button is NOT auto-focused; cancel button gets initial focus |
| Labels | All form inputs have associated labels (explicit via htmlFor or implicit wrapping) |
| Status change | Select/dropdown is keyboard navigable, status change announced via aria-live region |
