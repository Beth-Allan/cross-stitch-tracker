  <action>
    Fix the hydration mismatch in supply-catalog.tsx caused by reading localStorage in the useState initializer (lines 180-190).

    Root cause: `useState(() => { if (typeof window === "undefined") return DEFAULT_VIEWS; ... })` produces different values on server vs client, causing React hydration failure.

    Fix:
    1. Change the `viewModes` useState initializer to always return `DEFAULT_VIEWS` (no localStorage read):
       ```
       const [viewModes, setViewModes] = useState<Record<SupplyTab, ViewMode>>(DEFAULT_VIEWS);
       ```
    2. Add a `useEffect` that runs once on mount to restore saved preferences from localStorage:
       ```
       useEffect(() => {
         const restored = { ...DEFAULT_VIEWS };
         let changed = false;
         for (const tab of TAB_CONFIG) {
           const stored = localStorage.getItem(STORAGE_KEYS[tab.key]);
           if (stored === "grid" || stored === "table") {
             restored[tab.key] = stored;
             changed = true;
           }
         }
         if (changed) setViewModes(restored);
       }, []);
       ```
    3. The existing `setViewMode` callback that writes to localStorage stays as-is.

    This ensures server and client render identical initial HTML (DEFAULT_VIEWS), then client hydrates and applies saved preferences in a single post-hydration update.

    Update tests to verify the new pattern: initial render uses DEFAULT_VIEWS, then localStorage values apply after mount.
  </action>
