## Color Swatch Contract

Color swatches are the primary visual element of this phase. Prescriptive rules:

| Context | Size | Shape | Border Rule |
|---------|------|-------|-------------|
| Supply row (project tab) | 28px (w-7 h-7) | rounded-full | ring-1 ring-stone-200 dark:ring-stone-600 when luminance > 0.85 |
| Search dropdown result | 20px (w-5 h-5) | rounded-full | Same luminance rule |
| Grid view tile | 48px (w-12 h-12) | rounded-lg | Same luminance rule |
| Table view inline | 20px (w-5 h-5) | rounded-full | Same luminance rule |

**Luminance check function (from design reference):**
```
needsBorder(hex): boolean
  r = parseInt(hex[1..2], 16)
  g = parseInt(hex[3..4], 16)
  b = parseInt(hex[5..6], 16)
  return (0.299*r + 0.587*g + 0.114*b) / 255 > 0.85
```

All swatches use `style={{ backgroundColor: hexColor }}` with shadow-sm. This is CSS background-color from hex strings -- no canvas, no SVG, no color picker library.

**Source:** ProjectSuppliesTab.tsx needsBorder() helper, SupplyCatalog.tsx grid/table views

## Interaction Patterns

### Tab Switching (Supply Catalog)
- Tab click switches visible content immediately (no loading state)
- Active tab: emerald text + bottom border indicator
- Inactive tab: muted-foreground text, hover shows foreground

### Grid/Table View Toggle
- Two icon buttons (Grid3X3 for grid, List for table)
- Active view: emerald background tint, foreground icon
- Toggle persists to localStorage on change
- On mount: read localStorage, fallback to default (grid for threads, table for others)
- No flash of wrong view: render default on server, useEffect corrects client-side

### Inline Search-to-Add (Project Supplies Tab)
- Click "+ Add threads" opens search dropdown positioned below the section
- Search input auto-focuses
- Dropdown shows max 8 matching items with color swatch + brand code + name
- Already-linked supplies filtered out of results
- Click result: adds supply with default quantity 1, closes dropdown
- Click outside or Escape: closes dropdown
- "No matches" message when search has no results

### Editable Quantity (Project Supplies Tab)
- Quantity values ("Req:" and "Got:") are click-to-edit
- Click opens inline number input (w-12, focused, emerald border)
- Enter or blur: saves new value
- Escape: reverts to original value
- Invalid input (non-number, negative): ignored, reverts

### Supply Row Interactions
- Color swatch: click opens supply detail modal (cross-project usage)
- Code + name: click also opens detail modal
- Remove button (Trash2): opacity-0 by default, opacity-100 on row hover
- Row hover: subtle background change not needed (action visibility is the feedback)

### Shopping List Fulfillment
- "Mark acquired" on each item: updates acquired quantity to match required
- Item disappears from list after fulfillment (via revalidatePath)
- Project group disappears when all its supplies are fulfilled
- Toast: "DMC {code} marked as acquired" (sonner)

### Modal Forms (Supply/Brand/Fabric Create/Edit)
- Open via primary action button or edit icon
- Backdrop: bg-black/40 backdrop-blur-sm
- Modal: max-w-lg, rounded-2xl, shadow-2xl, scrollable content area
- Cancel button: ghost style (text-muted-foreground, hover:bg-muted)
- Save button: primary (bg-primary text-primary-foreground)
- Close via X button, Cancel, or backdrop click
- Validation errors shown inline below fields (text-destructive, text-sm)

### Sortable Tables
- Column header click toggles sort direction
- Active sort column: emerald text with arrow indicator
- Default sort direction: ascending on first click, toggle on subsequent
- Sort is client-side (data sets are small: ~500 threads max)

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Color swatch accessibility | Always display text label (brand + code) alongside swatch; never color-only identification |
| Keyboard navigation for search dropdown | Arrow keys navigate results, Enter selects, Escape closes |
| Editable quantity fields | Proper input type="number" with min constraint, keyboard Enter/Escape handling |
| Tab panel keyboard support | Tab key moves between tab triggers; Enter/Space activates; aria-selected on active tab |
| Modal focus trap | Focus stays within modal when open; Escape closes; return focus to trigger on close |
| Sortable table headers | Use button role on clickable headers; aria-sort attribute on active column |
| Remove button visibility | opacity-0 to opacity-100 on hover is mouse-only; ensure focus:opacity-100 for keyboard users |

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| Desktop (>=768px) | Full table views, side-by-side form fields, inline quantity editing |
| Mobile (<768px) | Grid view preferred for supplies (tiles stack naturally), table columns collapse to essential (swatch + code + quantities), modal forms go full-width, form fields stack vertically |
