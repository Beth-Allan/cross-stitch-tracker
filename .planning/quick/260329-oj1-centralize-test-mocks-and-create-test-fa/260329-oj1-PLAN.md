Create `src/__tests__/mocks/module-mocks.ts`:
- Import vi from vitest
- Export `setupNavigationMock()` - returns { mockPush } after calling vi.mock("next/navigation") with useRouter returning { push: mockPush }
- Export `setupChartActionsMock()` - returns { mockCreateChart, mockUpdateChart } after calling vi.mock("@/lib/actions/chart-actions") with delegating wrappers
- Export `setupServerActionMocks()` - calls vi.mock for designer-actions (createDesigner), genre-actions (createGenre), upload-actions (getPresignedUploadUrl) - all vi.fn() stubs
- Export `setupServerMocks()` - calls vi.mock for @/lib/auth (null session), @/lib/db (prisma with chart/project/designer/genre stubs), next/cache (revalidatePath stub) - returns { mockAuth, mockPrisma } for test override

IMPORTANT: vi.mock calls are hoisted by Vitest. The setup functions must be called at module top level in test files (not inside describe/beforeEach). The pattern is:
```typescript
// In module-mocks.ts - export the mock fns AND call vi.mock in same scope
export const mockPush = vi.fn();
vi.mock("next/navigation", () => ({
  useRouter: () => ({ push: mockPush }),
}));
```
Wait -- this won't work because vi.mock is hoisted per-file. Instead, use a different approach:
- Export ONLY the factory functions from factories.ts
- For module mocks, create snippet/template constants that document the standard mock patterns but do NOT try to share vi.mock calls (they must remain in each test file due to Vitest hoisting)
- Instead, export a `MOCK_PATTERNS` object documenting standard patterns (for human reference), and focus the real value on data factories

Revised approach for module-mocks.ts: Create helper functions that return mock objects (not vi.mock calls):
- `createMockPrisma()` - returns the prisma mock object shape
- `createMockRouter()` - returns { push: vi.fn(), ... }
These can be used inside vi.mock factory functions in test files.

Create `src/__tests__/mocks/index.ts`:
